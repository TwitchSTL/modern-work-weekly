Any service or container image embedding libcurl or golang.org/x/crypto/ssh needs immediate assessment.

5. **[Unified alert experience for Purview Insider Risk Management](https://techcommunity.microsoft.com/t5/microsoft-security-community/introducing-a-unified-alert-experience-for-microsoft-purview/ba-p/4530714)** — A consolidated alert queue, expanded user context, and connected case notes reduce analyst context-switching during IRM investigations. ## Visibility & Automation

- **[CVE-2026-9080: UAF after pause in socket callback](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-9080)** [GA] — Use-after-free in a socket callback pause condition in libcurl. Any service embedding libcurl is potentially affected; review your container base images and dependency manifests and prioritize patching.

- **[CVE-2026-10536: HTTP/2 stream-dependency tree UAF](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-10536)** [GA] — Use-after-free in HTTP/2 stream-dependency tree handling in libcurl. Exploitable in HTTP/2-enabled services; assess exposure in any API gateways, proxy services, or custom integrations using affected libcurl versions.

- **[CVE-2026-8926: Password leak with netrc and user in URL](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-8926)** [GA] — libcurl can leak credentials stored in netrc when a username is also present in the URL. Audit any automation scripts or pipeline agents using libcurl with netrc-based credential storage and rotate affected credentials after patching.

- **[CVE-2026-8286: Wrong STARTTLS connection reuse](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-8286)** [GA] — libcurl may incorrectly reuse a STARTTLS-upgraded connection for a request that should use a different security context. Services relying on STARTTLS for SMTP or IMAP integrations need to assess and patch.

- **[CVE-2026-8458: Wrong reuse for different services](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-8458)** [GA] — Connection reuse logic in libcurl can match connections across different services when it should not, potentially routing requests or leaking data to unintended endpoints. Patch and review any multi-service proxy or aggregation layers using libcurl.

- **[CVE-2026-8924: Trailing dot domain super cookie](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-8924)** [GA] — libcurl can set a "super cookie" scoped to a trailing-dot domain, potentially allowing a server to set cookies that affect requests to other domains. Relevant for any web-facing services or automation using libcurl for session-based authentication.

- **[CVE-2026-8932: Incomplete mTLS config matching in connection reuse](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-8932)** [GA] — libcurl may reuse a connection where mTLS client certificate configuration does not fully match, potentially presenting incorrect client identity to a server. Any service using mTLS for mutual authentication should treat this as high priority.

- **[CVE-2026-9545: Exposing HTTP/3 early data](https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-9545)** [GA] — HTTP/3 early data (0-RTT) can be exposed inappropriately in libcurl, potentially enabling replay attacks. Assess any HTTP/3-enabled endpoints or client integrations using libcurl.
