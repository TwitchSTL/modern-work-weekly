## Top 5 This Week

1. **Entra Agent ID Platform — GA.** Microsoft now has a first-class identity and authorization framework for AI agents using OAuth 2.0, MCP, and A2A. If you're deploying any Copilot Studio or custom agents in your tenant, this is the governance model you should be building around — start reviewing agent identity posture now.

2. **SAP SuccessFactors basic auth deprecation — November 2026 deadline.** SAP is pulling the plug on basic auth for SuccessFactors APIs. Microsoft Entra provisioning is introducing workload identity–based auth as the replacement. If you have Entra-to-SuccessFactors inbound provisioning, you need to plan and migrate before November 2026 or provisioning breaks.

3. **Dirty Frag Linux LPE — actively exploited.** A local privilege escalation vulnerability in Linux kernel networking components (esp4, esp6, rxrpc) is seeing limited in-the-wild exploitation. Any Linux endpoint or server reachable via SSH, web shell, or low-priv account is at risk. Patch Linux kernels now; Defender is detecting exploitation attempts but don't rely on that as your primary control.

4. **Entra Connect Sync → Cloud Sync migration announced.** Microsoft is formally beginning the transition away from on-premises Connect Sync to cloud-native Cloud Sync. No hard cutoff date yet, but the direction is unambiguous. Start your inventory of Connect Sync dependencies — custom sync rules, writeback configurations — and build a migration roadmap.

5. **macOS Platform SSO during ADE — GA.** Users get Entra ID access the moment they hit the desktop on a freshly enrolled Mac, with no post-enrollment SSO setup required. If you're running ADE for macOS and haven't configured Platform SSO, this is now the baseline expected experience.

## Endpoint Management

- **macOS Platform SSO during Automated Device Enrollment** `GA` — PSSO can now run during ADE registration, giving users immediate Entra ID resource access at first desktop. Requires pre-enrollment configuration. See: Configure PSSO during ADE for macOS.

- **Enhanced App Inventory for Windows** `GA` — Improved data freshness, richer app metadata, and new controls to scope which devices contribute to inventory. Windows-first; additional platforms coming. Useful for identifying outdated or risky software at scale.

- **EPM Support-Approved Elevations for All Device Users** `GA` — Endpoint Privilege Management now accepts support-approved elevation requests from any user of a device, not just the primary/enrolling user. Fixes a real gap for shared device scenarios.

- **Android Enterprise Credential Manager Control (Android 14+)** `GA` — Admins can now explicitly configure which apps act as credential providers (password autofill, passkey storage) on managed Android Enterprise devices. Navigate to Apps > Android > Configuration > Managed Devices. Note: Google Password Manager cannot act as credential provider on corporate-owned work profile devices — known limitation.

- **Userless ADE for visionOS and tvOS** `GA` — Apple Vision Pro and Apple TV can now be enrolled through Apple Business Manager/School Manager without a user. Manage alongside other Android/Apple devices in the admin center.

- **Ubuntu 26.04 LTS support added; Ubuntu 22.04 end of support August 2026** `Preview` — Ubuntu 26.04 LTS is now supported. Ubuntu 22.04 support ends August 2026. Enrolled 22.04 devices stay enrolled but you need to notify users to upgrade. Filter by OS version in Devices > All Devices > Linux to find your exposure.

- **Autopatch Update Risk Visibility Report** `GA` — New report classifies devices as Current, Exposed, or Critical and surfaces which policies are contributing to risk. Extends the Security Update Status dashboard. Good for prioritizing remediation conversations with device owners.

- **Edge 139 Security Baseline available in Intune** `GA` — Latest Edge browser security baseline is now available. Review and deploy alongside your existing Edge baseline profiles.

- **Windows 11 25H2 Security Baseline — GA** `GA` — New Windows 11 25H2 security baseline is available. Existing profiles do NOT auto-update. Admins must create a new profile or explicitly update existing ones to adopt the new settings, defaults, and retired settings.

- **Direct Android LOB App Management (COBO/COSU)** `GA` — Upload APK files directly to Intune and deploy to corporate-owned fully managed and dedicated devices without going through Managed Google Play. Streamlines workflows for apps that can't or shouldn't be published to the Play Store.

- **Android XR Device Management** `GA` — Android XR devices (including Android-based XR headsets) can now be enrolled via dedicated and fully managed modes. Deploy apps via Managed Google Play and apply compliance policies.

- **TeamViewer Connector Replacement** `GA` — New TeamViewer integration replaces the existing connector. **If you're on the old connector, you have 12 months to migrate before functionality breaks.** Begin migration planning now.

- **Intune Data Warehouse beta connector v1 (Power BI) — Retired** `Preview` — Reports created before November 2025 may still use the beta v1 connector. You must transition to connector v2 or the OData Feed connector. Reports created after November 2025 already use v2.

- **Apple DDM for Required LOB Apps (iOS/iPadOS 18+)** `GA` — Intune now supports Apple Declarative Device Management for required LOB apps on iOS/iPadOS 18 and later. Change management type to DDM in App information. Improves delivery efficiency and provides real-time app status.

- **macOS Recovery OS Password Policy** `GA` — Admins can configure and rotate a recovery OS password to prevent users from booting company-owned Macs into recovery mode and bypassing MDM. Password visible in per-setting status report with appropriate RBAC permission.

- **Remote Help Connectivity Improvement + New IME Log** `GA` — New endpoint added for Launch Remote Help. Update firewall rules accordingly. New `NotificationInfra.log` added to IME logs for tracking real-time communication channel notifications.

## Identity & Access

- **Microsoft Entra Agent ID Platform** `GA` — First-class identity framework for AI agents in enterprise environments. Supports OAuth 2.0, MCP, and A2A protocols. Developers can create and manage agent identities with enterprise-grade auth and governance. If you're building or deploying agents, this is your identity foundation. Start here.

- **Entra Connect Sync → Cloud Sync Migration** `Plan for Change` — Microsoft is formally transitioning away from on-premises Connect Sync to cloud-native Cloud Sync. No enforcement date announced yet, but posture is clear. Begin auditing Connect Sync deployments, custom rules, and writeback dependencies.

- **SAP SuccessFactors Provisioning: Basic Auth → Workload Identity** `Plan for Change` — **Hard deadline: November 2026 (SAP deprecation date).** Microsoft Entra provisioning will authenticate using workload identities and short-lived tokens instead of static credentials. If you have Entra → SuccessFactors inbound provisioning, migrate before SAP pulls the plug.

- **SCIM Provisioning Apps: Modern Authentication Migration** `Plan for Change` — Basic auth is being deprecated across SCIM provisioning integrations over the coming months. Timelines vary per application and will be communicated via monthly What's New articles and Message Center. Watch for your specific apps.

- **CBA on iOS + CBA as Second Factor** `GA` — Entra Certificate-Based Authentication is now GA on iOS. Eliminates unnecessary password/MFA prompts in native iOS apps. CBA can now be a second factor and system-preferred MFA method. Review your auth policies if you're using CBA.

- **Entra CBA Certificate Authority (CA) Scoping** `GA` — Tenant admins can now restrict specific CAs to defined user groups. Essential for environments with multiple PKI hierarchies — prevents over-broad certificate trust.

- **Entra CBA Issuer Hints** `GA` — Users on multi-certificate devices will only be prompted to select certificates that are trusted and valid for the tenant. Reduces sign-in errors. No behavior change for certificates themselves.

- **Enforce CA Policies (including MFA) on Every PIM Activation** `GA` — Reauthentication with Conditional Access is now GA for PIM role activations. Require MFA on every privileged role activation without workarounds. Implement this if you haven't already.

- **Configurable Token Lifetime Policies** `GA` — Admins can now customize access token, ID token, and SAML token lifetimes per application/service principal via policy. Use this to tighten token windows for sensitive apps or accommodate legacy systems that need longer lifetimes.

- **GSA iOS Client** `GA` — Global Secure Access client on iOS/iPadOS leverages existing MDE deployment — no additional agent required. Routes traffic through Microsoft SSE for M365, internet access, and private access.

- **GSA Network Content Filtering by File Type** `GA` — Block or monitor file transfers to GenAI and SaaS apps based on file type at the network layer. Useful for preventing data exfiltration through file upload to unsanctioned AI services.

- **GSA Cloud Firewall for Remote Networks** `GA` — 5-tuple (source IP, destination IP, protocol, source/destination port) based filtering for internet traffic from branch offices via GSA remote networks. Configurable from the admin center.

- **Workday Connector: Termination Lookahead for APAC/ANZ** `GA` — Fixes termination processing delays for workers in APAC and ANZ regions. Admins can enable termination lookahead to prefetch data and tailor deprovisioning logic. If you have Workday provisioning and operate in those regions, enable this.

- **Account Discovery for Connected Apps (Entra ID Governance)** `Preview` — Generates discovery reports from the provisioning experience to identify orphan accounts in connected applications that aren't assigned to the enterprise app in Entra. Good for access hygiene audits.

- **Entra External ID Federation with Workforce Identities** `Preview` — Users can now sign in to customer-facing (External ID) apps using their workforce Entra ID identity via standards-based federation. Reduces duplicate accounts for internal-facing consumer apps.

- **App-based Branding Themes** `Preview` — Assign different sign-in branding experiences to specific applications within a single tenant. Previously, branding was tenant-wide only.

- **MIM 2016 Service Pack 3** `GA` — SP3 updates SQL Server, SharePoint, and Exchange compatibility, and adds Azure SQL Database with managed identity auth support for the Synchronization Service. If you're still running MIM, apply SP3.

- **License Usage Page in Entra Admin Center** `GA` — Visibility into Entra ID P1, P2, and Suite license consumption. Shows feature usage (Conditional Access, risk-based CA) mapped to license type with 6-month trends. Useful for license right-sizing conversations.

- **Entitlement Management: Approver Visibility in My Access** `GA` — Requestors can now see the name and email of their approvers directly in My Access portal. Enabled by default for all members (non-guests). Controllable at tenant and access package level.

- **Universal Print Badge API** `Preview` — Map employee badge credentials (card/NFC) to user identities for secure print job release. No app, PIN, or device sign-in required. GA target: July 2026.

## Security & Compliance

- **Dirty Frag Linux LPE — Actively Exploited** `GA` — Local privilege escalation in Linux kernel networking (esp4, esp6, rxrpc). Allows unprivileged → root escalation. In-the-wild exploitation observed. Microsoft Defender provides detection but **patch your Linux kernels immediately**. Attack vectors include SSH, web shells, containers, and low-privileged accounts.

- **Kazuar Nation-State Botnet Analysis (Secret Blizzard / Russia)** `GA` — Microsoft published detailed analysis of Kazuar, attributed to Russian state actor Secret Blizzard. Kazuar has evolved from a traditional backdoor into a modular P2P botnet designed for persistent covert access. Read the full post to update your detection logic and threat model for espionage-focused intrusions.

- **Third-Party Compromise / Supply Chain Intrusion Investigation** `GA` — Microsoft Incident Response documented an attack that used legitimate administrative mechanisms to blend into routine operations. No loud exploits, no custom malware — just trusted tooling. Worth reviewing your third-party access policies and privileged account monitoring.

- **AI App Misconfigurations → RCE and Data Leaks** `GA` — Research post covering how exposed UIs, weak auth, and risky defaults in cloud-native AI apps on Kubernetes lead to RCE and data exfiltration. If you're running self-hosted AI workloads, audit your Kubernetes configurations against this research.

- **Purview DLP: Count of Labeled Documents Condition (Exchange Online)** `Preview` — New DLP condition: "Count of labeled documents is greater than [threshold]." Triggers when an email contains more than N attachments with a matching sensitivity label. Enables detection of bulk sensitive data exfiltration that individual file checks miss.

- **Purview Data Lifecycle Management Billing Change** `GA` — DLM billing is moving to volume-based (per GB retained) for non-Microsoft 365 generative AI prompts and responses. Rate: $0.25/GB/month. If you have DLM policies covering AI data, model your cost exposure now.

- **Purview Endpoint DLP: Copilot+ PC Recall Snapshot Coverage** `Preview` — Endpoint DLP can now evaluate whether Recall snapshots on Copilot+ PCs contain content restricted by sensitivity labels or SITs, and block capture accordingly. Requires coordinated policy authoring between Purview and Intune admins. GA was targeted for April 2026 — verify current rollout status in your tenant.

- **Purview DLP Additional Diagnostics (SPO, EXO, Auto-labeling)** `Preview` — New diagnostic tools for DLP SharePoint, DLP Exchange Online, and auto-labeling on SPO/OneDrive. Helps identify which SITs exist in a document, what labels are applied, and which policies triggered. GA targeted February 2026 — check availability in your tenant.
