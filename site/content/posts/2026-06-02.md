## Top 5

1. **Teams Live Events retirement is June 30, 2026 — 4 weeks away.** New live event scheduling is blocked at that date; events already scheduled run through February 28, 2027. If your org runs recurring large-scale broadcasts using Teams live events, you need to migrate to the new Teams events experience (public preview) *now*. Graph API integrations that create live events must be re-platformed before the deadline.

2. **Exchange ActiveSync Certificate-Based Authentication (CBA) to Exchange Online is deprecated immediately.** New tenants are blocked from the legacy CBA flow with immediate effect; all remaining tenants must migrate EAS CBA through Entra ID before end of 2026. Any mobile device or MDM profile using direct certificate auth to Exchange Online will stop working. Audit your EAS profiles in Intune and validate you're routing through Entra ID CBA.

3. **Agent 365 is GA — your control plane for AI agent sprawl.** Microsoft Agent 365 is generally available and provides observe/govern/secure capabilities for agents and their interactions. If your org is deploying Copilot Studio agents at scale, this is the governance layer you've been waiting for. Pair it with the updated Administering and Governing Agents whitepaper v3.2 and the Purview DSPM + AI Security Dashboard architecture to close the visibility gap.

4. **Phishing-resistant MFA for Linux desktops is now GA.** Microsoft Entra now extends phish-resistant MFA to Linux via the Microsoft identity broker, closing a long-standing cross-platform gap. If you have Linux workloads (dev environments, servers with interactive sign-ins) that currently fall back to weaker auth, this removes the last excuse — update your Conditional Access policies to enforce phishing-resistant MFA across all platforms.

5. **CVE-2026-42897 — on-premises Exchange OWA XSS requires immediate mitigation.** All on-prem Exchange versions (2016, 2019, SE) are affected; Exchange Online is not. An attacker can execute arbitrary JavaScript in a user's browser via a crafted email opened in OWA. Microsoft has published a mitigation — apply it immediately if you're running on-prem Exchange and haven't deployed the fix yet.

## Identity

- **[What's New in Microsoft Entra: June 2026](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/what-s-new-in-microsoft-entra-june-2026/ba-p/4517885)** [GA] — Phish-resistant MFA for Linux desktops via the Microsoft identity broker is the headline GA item; if you have Linux in scope for Zero Trust, update your CA policies immediately. The June roundup also covers Tenant Configuration Management API GA, account discovery for app governance, and multiple preview graduations from Q1/Q2.

- **[Phishing-Resistant MFA for Linux Desktops — GA](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/what-s-new-in-microsoft-entra-june-2026/ba-p/4517885)** [GA] — Linux desktops can now authenticate using phishing-resistant credentials through the Microsoft identity broker, reaching parity with Windows and macOS. This closes the last major platform gap in enforcing phish-resistant MFA at Conditional Access. Audit any CA policies with platform exclusions for Linux and tighten them.

- **[Entra Tenant Configuration Management APIs — GA](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/tenant-configuration-management-apis-are-now-generally-available/ba-p/4513157)** [GA] — The Tenant Configuration Management APIs are production-ready, providing a programmatic control plane for enforcing consistent configuration across multi-tenant environments. If you manage multiple tenants (MSP, enterprise, or acquired-company scenarios), this is your baseline for automated posture enforcement and drift detection.

- **[Find Shadow Tenants with Microsoft Entra Tenant Governance](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/find-shadow-tenants-and-reduce-risk-fast-with-microsoft-entra/ba-p/4521996)** [GA] — Tenant Governance now discovers related tenants via B2B collaboration signals, multitenant app registrations, and shared billing — surfacing tenants that fell outside central IT oversight. Shadow tenants are a real risk vector; run discovery now and establish governance for anything that surfaces.

- **[Platform SSO During Automated Device Enrollment — macOS GA](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/platform-sso-during-automated-device-enrollment-is-now-generally/ba-p/4436813)** [GA] — Platform SSO (PSSO) now triggers automatically during ADE enrollment on macOS, so device registration and SSO setup happen without extra steps for IT or end users. This removes a common manual touchpoint in macOS provisioning workflows; validate your ADE profiles in Intune to ensure PSSO is configured to take advantage of this.

- **[Account Discovery for Application Access Governance](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/you-can-t-govern-what-you-can-t-see-closing-the-identity/ba-p/4507464)** [Preview] — Account discovery in Entra ID Governance identifies local and orphaned accounts in connected applications that were created outside modern governance workflows, mapping them to Entra users. The GA target is June 2026 — if you're onboarding SaaS apps to Entra Governance, enroll in preview now to start closing your access visibility gap before GA.

- **[License Usage Insights in Microsoft Entra — GA](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/now-generally-available-license-usage-insights-in-microsoft/ba-p/4507463)** [GA] — Redesigned license usage insights in the Entra admin center now show which identity protections are active per user and where gaps exist. Useful for identifying under-deployed controls and making the case for expanding P2 features to uncovered users.

- **[Microsoft Identity Manager 2016 SP3 — GA](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/microsoft-identity-manager-2016-sp3-now-available-enhanced/ba-p/4519489)** [GA] — MIM 2016 SP3 updates compatibility with current SQL Server, SharePoint, and Exchange versions and improves stability for organizations that can't yet migrate fully to cloud identity. If you're still running MIM, apply SP3 to maintain supportability; also use this as a forcing function to audit your remaining MIM-dependent scenarios against Entra's cloud capabilities.

- **[Retirement of Direct Exchange ActiveSync Certificate-Based Authentication](https://techcommunity.microsoft.com/t5/exchange-team-blog/retirement-of-direct-exchange-activesync-certificate-based/ba-p/4517896)** [GA] — EAS CBA directly to Exchange Online is deprecated; new tenants are blocked immediately and all tenants must migrate to Entra ID-brokered CBA before end of 2026. Run `Get-MobileDeviceStatistics` and audit Intune device configuration profiles for any EAS certificate profiles pointing directly at Exchange Online.

- **[Passkeys Aren't the Finish Line: Eliminating Fallbacks and Fixing Recovery](https://techcommunity.microsoft.com/t5/microsoft-entra-blog/passkeys-aren-t-the-finish-line-eliminating-fallbacks-and-fixing/ba-p/3627345)** [Broad] — Microsoft's World Passkey Day post makes the architectural point that passkey deployments are only as strong as their fallback and recovery paths. AI-driven phishing click-through rates have hit 54% — if your passkey rollout still allows SMS or knowledge-based recovery, you haven't eliminated the attack surface.

- **[Hardening Administrative Actions on Windows](https://techcommunity.microsoft.com/t5/windows-it-pro-blog/hardening-administrative-actions-what-it-pros-need-to-know/ba-p/4503956)** [GA] — Windows now detects and blocks authentication attempts between machines sharing duplicate SIDs, breaking a privilege escalation path that could be exploited via cloned or misconfigured devices. This is a breaking change if your environment has cloned device images with duplicate SIDs — inventory and remediate before the hardening causes unexpected auth failures.
